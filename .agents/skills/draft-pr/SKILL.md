# draft-pr

Use this skill when the issue branch is ready for review.

Workflow:
1. Confirm the branch is issue-specific and not the default branch.
2. Summarize purpose, changed files, verification outcome, and known risks.
3. Ensure the workpad contains branch, PR, verification, artifacts, and blockers.
4. Create or update a draft PR only after verification artifacts are complete.
5. Set the issue state to `Human Review` after the draft PR exists.
