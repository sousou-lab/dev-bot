特に近いのは、単一の control plane を持つこと、workspace を issue 単位で分離すること、DB ではなく filesystem artifacts と tracker state を軸に復旧すること、そして policy を repo に置くことです。IssueScheduler は Ready / Rework だけを dispatch 対象にし、In Progress は reconciliation、Merging は別処理として扱っています。Orchestrator も item key と thread id で重複起動を防ぐ薄い実行レイヤとして機能していて、設計意図は明確です。

改善の優先順位で、いちばん効くのは WORKFLOW.md を本当に typed contract 化することです。いま確認した workflow_loader.py は、WORKFLOW.md を読み、front matter を yaml.safe_load して raw_text と contract_body を返すかなり軽い実装です。これは悪くないのですが、Symphony 的な typed config layer と比べると、validation、default、last-known-good reload、dynamic reload までは持っていません。ここを WorkflowConfig の明示スキーマ、validation error の型、reload 時の差分適用、invalid reload 時の fail-open ではなく “last good config 維持” に寄せると、設計が一段安定します。少なくとも、loader は薄く、config layer は厚くにした方がよいです。

モデル設定も整理した方がいいです。WORKFLOW.md では Codex 側モデルが gpt-5.4 になっている一方で、CodexRunner のデフォルト値はまだ gpt-5-codex です。最新の Codex docs では、Codex ではまず gpt-5.4 を勧めるとされていて、2026-03-05 の changelog でも、より強い tool use、experimental な 1M context、native computer-use が入った推奨モデルとして位置づけられています。実際に runtime 側で WORKFLOW.md の model を必ず反映できているなら良いのですが、契約値と runner default がズレている状態は早めに潰すべきです。

implementation lane は、複数 editor を同じ worktree に走らせないのが原則です。速度と精度を両立したいなら、標準形は「1 editor + 1 verifier + 1 reviewer」です。editor だけが書き込み、verifier と reviewer は read-only で diff・tests・scope drift を見る。複数 editor を使うなら、Codex app の発想どおり agent ごとに別 worktree を切って candidate patch を作り、最後に 1つの integrator が採用・統合するべきです。Codex app 自体が parallel threads と built-in worktree support を前提にしていて、review pane で inline feedback、stage/revert、diff review を行う設計になっています。dev-bot に取り込むなら、PR 前の “candidate patch tournament” と “line-anchored review feedback” が効きます。

もし本格的に multi-agent implementation をやるなら、Codex を app-server のさらに上で束ねるより、Codex を MCP server として expose して上位オーケストレータから呼ぶ構成も検討価値があります。OpenAI は Codex を MCP server として使い、Agents SDK から multi-agent development workflow を組むガイドを出しています。つまり、dev-bot の Python orchestrator の上位に「editor agent」「review agent」「merge agent」を置くとき、下位実行単位として Codex をツール化する形です。これは “複数 editor を走らせたいが protocol を自前で全部背負いたくない” 場面で相性がいいです。

harness engineering で一番相性がいいのは、repo knowledge を system of record にする発想です。OpenAI の記事では、大きすぎる AGENTS.md は失敗しやすく、短い AGENTS.md を目次にして、実体は docs/ 配下に構造化して置く方がよいとしています。あなたの repo はすでに .agents/skills、.claude、AGENTS.md、WORKFLOW.md を持ち、docs/references や docs/exec-plans/active もあります。かなり良い土台です。ここからさらに、docs/QUALITY_SCORE.md、SECURITY.md、RELIABILITY.md、ARCH_DECISIONS/、tech-debt tracker みたいな “agent が迷ったときの判断根拠” を増やすと、planning も implementation も安定します。

同時に、harness engineering の “entropy and garbage collection” も取り込むべきです。OpenAI は agent-generated code では drift が必ず起きるので、継続的な garbage collection が必要だと書いています。dev-bot なら、issue ベースの本流とは別に、低リスクな background hygiene run を設けるとよいです。たとえば「typing drift の是正」「stale skill の改善」「verification profile の棚卸し」「docs とコードの乖離検出」です。Codex app の automations も、background worktree で定期ジョブを回す設計になっています。これは本流の issue delivery を速くする間接施策です。

テストについては、いまの単体・統合テスト群はかなり充実しています。test_orchestrator.py、test_pipeline.py、test_runner_protocol.py、test_state_store.py、test_workflow_and_proof.py、test_workspace_manager.py、integration テストまで見えます。なので不足しているのは “agent layer をコードで全部再現するテスト” ではなく、captured run を採点する eval harness です。OpenAI の eval 記事が言うように、agent eval は「prompt → trace + artifacts → small set of checks → score」で良い。つまり、plan.json の妥当性、changed_files.json の scope、verification.json の整合、実チェック通過、不要変更、protected path touch などを grader 化すれば、重い multi-agent E2E を増やさずに改善を回せます。さらに Claude Code は OTel 監視も出せるので、planning lane の token / tool / latency を採るのも有効です。

1. WORKFLOW.md を「読むだけ」から「型付き契約」に上げる

2. planning lane を PlannerCommittee 化する

plan品質を優先するなら、いちばん投資対効果が高いのはここです。Anthropic は multi-agent research で、並列探索が効く問題では lead + subagents が単独 agent を大きく上回る一方、coding は research ほど並列化しやすくないとも書いています。だから multi-agent を入れる場所は implementation 本体より、planning / exploration / risk discovery / test design が先です。

構成はこうです。

repo_explorer
影響ファイル候補、既存パターン、アーキ依存、類似実装を拾う。

risk_test_planner
失敗モード、 acceptance criteria の穴、必要テスト、回帰点を洗う。

constraint_scope_checker
out-of-scope、保護パス、 risky migration、必要承認を洗う。

上の3本を統合し、唯一の plan.json / test_plan.json / verification_plan.json に落とす。

3. PlanContextBuilder を入れて、role ごとに渡す文脈を絞る

のような形で十分です。

補足すると、Canly の記事は「5つの独立したレビューエージェント」を紹介していますが、現在の公式 plugin README は 4 parallel review agents + confidence scoring を記述しています。なので、実装の基準は Canly の記事ではなく 公式 README / official docs に寄せるのが安全です。Canly の実測コストは参考になりますが、構成数のソースとしては公式の方を採ります。

7. debug 専用の debug_bundle / incident_bundle を入れる

Dinii の記事は anecdotal ですが、示唆はかなり良いです。彼らは、AI が本番 DB / logs / Sentry を直接見られるようになったことと、仮説検証を並列化したことで debug 品質が大きく上がったと書いています。dev-bot にそのまま prod 直アクセスを渡す必要はありませんが、read-only で匿名化済みの debug bundle を agent に渡せるようにするのは有効です。

つまり、本番に Bash させるのではなく、incident_bundle/ に

stack trace

sanitized logs

relevant DB rows dump

feature flags

request/response samples

Sentry issue snapshot

を詰めて、planner / debugger / reviewer がそれを見る構成です。高信頼環境だけ、限定 MCP / dynamic tool で read-only 参照を許す。Claude Agent SDK は permission modes / allow-deny / hooks を持ち、Codex app-server も file/network approvals や dynamic tool calls を持つので、**「直アクセス」ではなく「狭い read-only tool」**として切るのがよいです。

8. synthetic eval を先に作る、は正しい

この方針は賛成です。Anthropic の evals 記事は、20–50 の簡単なタスクから始めれば十分、むしろ早く始める方がよいとかなり明確に言っています。OpenAI / Anthropic の両方が、agent 開発では trace と artifacts を見ながら小さく回すことを重視しています。

最初の dataset は 4 種でいいです。

planning-basic
issue から plan.json / test_plan.json を作る

planning-risky
scope trap / hidden constraint / protected path を含む

review-seeded-bugs
PR diff に既知バグを 1–3 個埋める

debug-bundle
コードだけでは解けず、bundle を読まないと解けない

採点は

planning: completeness, scope discipline, test adequacy, risk coverage

review: precision, recall, localization, severity calibration

implementation: pass@1, pass^k, latency, unnecessary file touches, protected-path violations
で回せば十分です。customer-facing reliability を見るなら pass^k を併記します。

3つ目は、Codex 実行バックエンドを抽象化することです。OpenAI の最新 docs では、app-server は rich client 向け、CI や automation には Codex SDK を使うのが本筋です。ただ、app-server は turn/steer、thread/resume、outputSchema、approval イベントなど、今の dev-bot に欲しい deep integration 機能が豊富です。だから今すぐ全面移行ではなく、ExecutionBackend を切って app-server を既定、SDK を将来差し替え可能にするのが一番安全です。

4つ目は、review を独立 lane にすることです。Anthropic の公式 Code Review は、複数 agent が diff と周辺コードを並列に見て、verification で false positive を落とし、severity を付けて inline comment 化する流れです。公式 action 側も、tentative comment をそのまま出さず、分類してから投稿する入力を持っています。dev-bot でもこれを真似て、finding は即投稿せず、verifier と ranker を通してから出すのが良いです。

5つ目は、synthetic eval を最初から独立基盤にすることです。Anthropic は「agent を評価するなら harness と model の組み合わせを評価している」と明言していて、まず小さな eval suite を作ることを勧めています。あなたが言う「synthetic で立ち上げてから replay」に私は賛成です。

2. まず作るべき canonical policy source

手書きで増やすのは次の3ファイルです。

docs/policy/agent-rules.md
docs/policy/review-rules.md
docs/policy/test-rules.md

そこから build step で、次を生成します。

AGENTS.md      # Codex / OpenAI向け
CLAUDE.md      # Claude Code / Agent SDK向け
REVIEW.md      # review専用

この分離はかなり重要です。Codex は AGENTS.md を run 前に読み、Claude Code Review は CLAUDE.md と REVIEW.md を additive に読みます。REVIEW.md は「何を flag すべきか」「何を skip すべきか」を review 専用に切り出せるので、general instructions を汚しません。

生成する理由は、mixed stack だと drift が即起きるからです。しかも OpenAI 側は「巨大な AGENTS.md は失敗しやすい」、Anthropic 側は「review-specific guidance は REVIEW.md に分けろ」と言っています。だから canonical source を docs/policy/ に置いて、surface ごとの最適化だけ generator に持たせるのが一番きれいです。

3. PlannerCommittee の具体形

planner はこの4役で始めるのがよいです。

repo_explorer        # 影響範囲・既存パターン・類似実装
risk_test_planner    # failure mode・必要テスト・回帰点
constraint_checker   # out_of_scope・保護パス・承認要件
plan_merger          # canonical plan/test/verification を確定

Anthropic の Python Agent SDK では、query() は毎回 fresh session の one-off task 向け、ClaudeSDKClient は継続会話向けです。なので committee の前3役は query()、plan差し戻しや plan revision thread だけ ClaudeSDKClient で十分です。しかも Agent SDK は output_format に JSON Schema を渡せるので、committee の出力を最初から schema で固められます。

ここで1つ実務上かなり大事なのは、Python SDK には dontAsk がないことです。TypeScript なら allowedTools + permissionMode: "dontAsk" で headless lock-down がしやすいのですが、Python では明示 deny と hooks で詰める必要があります。plan layer を Python のまま行くなら、allowed_tools だけに期待せず、disallowed_tools と hooks を厚くしてください。TypeScript sidecar を許容できるなら、planning か reviewer の一部だけ TS に寄せる価値はあります。

さらに、Claude subagents は parent permissions を継承しつつ tool を狭められ、inline MCP server を subagent だけに閉じ込めることもできます。これは planning / review でかなり便利です。たとえば history_reviewer だけに Git history MCP を与え、親会話には載せない、のような設計ができます。なお agent teams は設定フラグで有効化する実験機能なので、prod の主戦力は explicit subagents / SDK orchestration にして、agent teams 依存はまだ避けるのが無難です。

4. plan品質を上げる gate

plan 承認前に、PlanQualityGate を必ず通します。
通す条件は、最低でもこの6つです。

acceptance criteria が issue body / workpad と 1対1で追える

out_of_scope が明示されている

candidate_files がある

acceptance criterion ごとに test mapping がある

risky point ごとに mitigation がある

verification profile が解決済み

これは単なる prompt 工夫ではなく、artifact contract を機械判定する話です。Anthropic の context engineering と long-running harness は、プロンプトを盛るより、何を残し何を次セッションに渡すかを構造化する方が効くと説明しています。

5. implementation は single-writer を標準にする

通常モードはこうです。

1 editor (Codex)
+ 1 verifier (read-mostly)
+ 1 reviewer (read-only)
+ 1 test_maker (optional, read/write only on tests)

Codex の multi-agent docs は、parallel agents は exploration / tests / triage / summarization に向く一方、parallel write-heavy workflows は conflict と coordination overhead が増えると明確に言っています。だから通常は single-writer がよいです。

ただし、あなたは同一 issue で複数 worktree を許容できるので、candidate mode は入れる価値があります。Codex app / worktrees / automations は、parallel task を dedicated worktree で分離する設計をかなり明確に採っています。なので dev-bot でも、難題だけ worktree を2本切る設計は理にかなっています。

私なら identity はこう拡張します。

issue_key
run_id
candidate_id   # primary | alt1 | alt2

ただし、scheduler 目線では active run は1つ のままです。candidate はあくまで run の子です。Ready/Rework -> In Progress の gate も崩しません。candidate を複数出しても State=In Progress は1本です。これは今の ARCHITECTURE.md の原理と整合します。

candidate mode を起動する条件は、最初はこれで十分です。

- rework_count >= 1
- planner_confidence < 0.75
- risk_score >= 0.7
- expected_touched_files >= 8
- architecture decision が 2案以上ある

そして winner selection は、verifier + reviewer + integrator で決めます。editor 同士に merge させない方が安定します。

6. Codex 実行バックエンドの方針

ここは結論が明確です。いまは app-server 維持、ただし SDK backend を切れるようにする、です。
理由は2つあります。

1つ目。OpenAI は app-server を rich client 向け、Codex SDK を CI/CD や programmatic control 向けと書いています。dev-bot は daemon なので、将来的には SDK 向きです。

2つ目。今すぐは app-server の方が欲しい primitives が揃っています。turn/steer、thread/resume、outputSchema、serviceName、approval イベント、dynamic tools、turn/interrupt などです。dev-bot が Discord / GitHub / workpad と細かく同期するには、この粒度が便利です。

なので、execution_backend.py だけは最初から interface にします。

class ExecutionBackend(Protocol):
    async def start_run(self, spec: RunSpec) -> RunHandle: ...
    async def steer(self, handle: RunHandle, message: str) -> None: ...
    async def interrupt(self, handle: RunHandle) -> None: ...
    async def resume_same_run(self, handle: RunHandle) -> RunHandle: ...
    async def collect_outputs(self, handle: RunHandle) -> RunArtifacts: ...

実装はまず CodexAppServerBackend。その後、Node 18+ を sidecar として許せるなら CodexSDKSidecarBackend を足します。Codex SDK は TypeScript library なので、Python orchestrator から直に使うより sidecar の方が自然です。

さらに、abort / reconcile では いきなり process kill しないで、まず turn/interrupt、それがだめなら hard kill、に変えると綺麗です。app-server は in-flight turn の interruption を持っています。

7. 最新機能の取り込み先

いま取り込む価値が高いのは、OpenAI 側ではこの4つです。
gpt-5.4 統一、outputSchema、turn/steer、serviceName です。最新 docs では Codex はまず gpt-5.4 を使うのが推奨で、長い multi-step workflow、tool use、1M context、compaction、computer use を持つ mainline model とされています。

Anthropic 側では、subagents、output_format JSON Schema、project settings 読み込み、permissions/hooks、REVIEW.md、OTel が優先です。Claude Code Review は multiple agents + verification + severity という構造で、REVIEW.md に review-only rules を置けます。Agent SDK は output_format に JSON Schema を渡せ、.claude/settings.json を読むには setting_sources=["project"] が必要です。Claude Code 自体も OTel export をサポートしています。

10. review 層の具体アルゴリズム

review はこの順で動かします。

これは Anthropic の Code Review の「parallel analysis → verification → severity ranking」とほぼ同じ構造です。違うのは、managed service ではなくlocal harness に落とすことです。これなら cost と control を持てます。Anthropic の managed Code Review は平均20分、平均$15–25 とされており、外部の Canly 記事もコストの高さを報告しています。だから dev-bot では自前の軽量版が現実的です。

ここで取り込むべき小技は、claude-code-action の classify_inline_comments 発想です。つまり、reviewer subagent の tentative findings はその場で PR に投げず、buffer して classify / verify 後にだけ投稿します。これで test probe や中間思考の混入を防げます。

そして repo root に REVIEW.md を置きます。内容は、

skip するもの
だけに絞ります。これは Anthropic の公式が勧めている使い方そのままです。

5. PlannerCommittee の責務

planning は4役で十分です。

repo_explorer

risk_test_planner

Anthropic の Python SDK では query() は毎回 fresh session、ClaudeSDKClient は継続会話向けです。なので初回の探索 3役は query()、差し戻しや再承認で plan を育てるときだけ ClaudeSDKClient を使うのが自然です。

擬似コードはこうです。

async def build_plan(issue_ctx: IssueContext) -> PlanBundle:
    ctx = CommitteeContextBuilder.from_issue(issue_ctx)

    repo_task = run_repo_explorer(ctx.repo_pack)
    risk_task = run_risk_test_planner(ctx.risk_pack)
    constraint_task = run_constraint_checker(ctx.constraint_pack)

    repo_out, risk_out, constraint_out = await asyncio.gather(
        repo_task, risk_task, constraint_task
    )

    merged = await run_plan_merger(
        ctx.merge_pack(
            repo_out=repo_out,
            risk_out=risk_out,
            constraint_out=constraint_out,
        )
    )

    PlanQualityGate.validate_or_raise(merged)
    return merged

各 role の tool surface は初期はかなり狭くします。Python SDK では dontAsk がないので、allowed_tools だけでは不十分で、disallowed_tools と hooks を併用した方が安全です。あなたの方針どおり、強い tool は role ごとに閉じ込めます。Anthropic docs でも Python では dontAsk がなく、TypeScript only と明記されています。

初期設定はこうでよいです。

ClaudeAgentOptions(

    setting_sources=["project"],
    allowed_tools=["Read", "Grep", "Glob"],
    disallowed_tools=["Write", "Edit", "Bash", "WebSearch", "WebFetch"],
    output_format={"type": "json_schema", "schema": PLAN_SCHEMA},
)

setting_sources=["project"] を必ず入れるのは、.claude/settings.json と CLAUDE.md を確実に効かせるためです。

async def review_run(run_ctx: ReviewContext) -> ReviewBundle:
    raw = await asyncio.gather(
        diff_reviewer(run_ctx),
        history_reviewer(run_ctx),
        contract_reviewer(run_ctx),
        test_reviewer(run_ctx),
    )
    merged = dedupe_findings(raw)
    verified = await evidence_verifier(run_ctx, merged)
    ranked = await ranker(run_ctx, verified)

    postable = [
        f for f in ranked.findings
        if f.confidence >= 0.80 and f.verifier_status == "confirmed"
    ]
    return ReviewBundle(all_findings=ranked.findings, postable=postable)

Canly の記事は「ローカル plugin 版は 5 agent・confidence 80 以上を既定」と紹介していて、その閾値感は参考になります。ただし、contract として寄せる先は公式 docs の「parallel review → verification → severity ranking」です。Canly の数値は運用チューニングの参考、構造は公式設計を採用がよいです。

from __future__ import annotations

from pathlib import Path
import hashlib
import yaml

ROOT = Path(__file__).resolve().parents[1]

def load_manifest() -> dict:
    return yaml.safe_load((ROOT / "docs/policy/manifest.yaml").read_text(encoding="utf-8"))

def render_output(name: str, sources: list[str]) -> str:
    blocks = []
    for rel in sources:
        text = (ROOT / rel).read_text(encoding="utf-8").rstrip()
        blocks.append(text)
    body = "\n\n".join(blocks).rstrip() + "\n"
    checksum = hashlib.sha256(body.encode("utf-8")).hexdigest()
    header = [
        "<!-- GENERATED FILE. DO NOT EDIT.",
        f"output: {name}",
        "sources:",
        *[f"  - {s}" for s in sources],
        f"checksum: sha256:{checksum}",
        "-->",
        "",
    ]
    return "\n".join(header) + body

def main() -> None:
    manifest = load_manifest()
    for output_name, sources in manifest["outputs"].items():
        out = render_output(output_name, sources)
        (ROOT / output_name).write_text(out, encoding="utf-8")

if __name__ == "__main__":
    main()

from __future__ import annotations

from typing import Literal
from pydantic import BaseModel, Field, model_validator

PlannerTool = Literal["Read", "Grep", "Glob", "Edit", "Write", "Bash", "WebSearch", "WebFetch"]

class CommitteeRoleConfig(BaseModel):
    mode: Literal["query", "session"]
    allowed_tools: list[PlannerTool] = Field(default_factory=list)
    disallowed_tools: list[PlannerTool] = Field(default_factory=list)
    output_schema: str

class PlanningGates(BaseModel):
    require_out_of_scope: bool = True
    require_candidate_files: bool = True
    require_test_mapping: bool = True
    require_verification_profile: bool = True
    require_design_branches: bool = True

class PlanningCommitteeConfig(BaseModel):
    roles: dict[str, CommitteeRoleConfig]

class PlanningConfig(BaseModel):
    provider: Literal["claude-agent-sdk"]
    enabled: bool = True
    mode: Literal["committee"] = "committee"
    cwd_source: Literal["plan_workspace"] = "plan_workspace"
    max_turns: int = 4
    timeout_seconds: int = 300
    settings_sources: list[Literal["project", "user", "local"]] = Field(default_factory=lambda: ["project"])
    committee: PlanningCommitteeConfig
    gates: PlanningGates = Field(default_factory=PlanningGates)

class CandidateModeTriggers(BaseModel):
    rework_count_gte: int = 1
    planner_confidence_lt: float = 0.75
    require_clear_design_branches: bool = True

class CandidateModeConfig(BaseModel):

PlanQualityGate は prompt ではなく機械判定です。

class PlanQualityGate:
    @staticmethod
    def validate_or_raise(plan: PlanV2) -> None:
        assert plan.acceptance_criteria
        assert "out_of_scope" in plan.model_fields_set
        assert plan.candidate_files
        assert plan.verification_profile
        assert plan.test_mapping
        assert {m["criterion"] for m in plan.test_mapping} >= set(plan.acceptance_criteria)

        if plan.planner_confidence < 0.75:
            return  # pass plan, but candidate mode may trigger later

        if not plan.design_branches:
            raise ValueError("design_branches required")

        if needs_branch_comparison(plan) and len(plan.design_branches) < 2:
            raise ValueError("clear design branch analysis required")

OpenAI の multi-agent docs は、main conversation を exploration notes や test logs で汚さず、parallel read-heavy workers の summary だけ返すのを勧めています。これは committee 設計とそのまま一致します。

async def review_run(ctx: ReviewContext) -> ReviewBundle:
    raw_findings = await asyncio.gather(
        diff_reviewer(ctx),
        history_reviewer(ctx),
        contract_reviewer(ctx),
        test_reviewer(ctx),
    )

    merged = dedupe(raw_findings)
    verified = await evidence_verifier(ctx, merged)
    ranked = await ranker(ctx, verified)

    postable = [
        f for f in ranked.findings
        if f.confidence >= ctx.thresholds.min_confidence_to_report
        and f.verifier_status == "confirmed"
    ]

    return ReviewBundle(
        all_findings=ranked.findings,
        postable=postable,
    )

def select_winner(primary: CandidateResult, alt1: CandidateResult) -> str:
    scored = []
    for c in [primary, alt1]:
        score = 0
        score += 100 if c.verification.hard_checks_pass else -1000
        score -= 50 * c.review.high_count
        score -= 20 * c.review.medium_count
        score -= 10 * c.scope.unexpected_file_count
        score += 10 * c.summary.plan_alignment_score
        scored.append((score, c.candidate_id))
    return max(scored)[1]

from __future__ import annotations

from pathlib import Path
import hashlib
import yaml

ROOT = Path(__file__).resolve().parents[1]


def load_manifest() -> dict:
    path = ROOT / "docs/policy/manifest.yaml"
    return yaml.safe_load(path.read_text(encoding="utf-8"))


def build_output(output_name: str, sources: list[str]) -> str:
    chunks: list[str] = []
    for rel in sources:
        text = (ROOT / rel).read_text(encoding="utf-8").rstrip()
        chunks.append(text)

    body = "\n\n".join(chunks).rstrip() + "\n"
    checksum = hashlib.sha256(body.encode("utf-8")).hexdigest()

    header = [
        "<!-- GENERATED FILE. DO NOT EDIT.",
        f"output: {output_name}",
        "sources:",
        *[f"  - {src}" for src in sources],
        f"checksum: sha256:{checksum}",
        "-->",
        "",
    ]
    return "\n".join(header) + body


def main() -> None:
    manifest = load_manifest()
    outputs = manifest["outputs"]
    for output_name, sources in outputs.items():
        rendered = build_output(output_name, sources)
        (ROOT / output_name).write_text(rendered, encoding="utf-8")


if __name__ == "__main__":
    main()

    risk: RiskTestPlanV1
    constraint: ConstraintReportV1
    merged: PlanV2


class PlannerCommittee:
    def __init__(
        self,
        repo_explorer: RepoExplorer,
        risk_test_planner: RiskTestPlanner,
        constraint_checker: ConstraintChecker,
        merger: PlanMerger,
    ) -> None:
        self.repo_explorer = repo_explorer
        self.risk_test_planner = risk_test_planner
        self.constraint_checker = constraint_checker
        self.merger = merger

        repo_task = self.repo_explorer.run(ctx.repo_pack)
        risk_task = self.risk_test_planner.run(ctx.risk_pack)
        constraint_task = self.constraint_checker.run(ctx.constraint_pack)

        repo_out, risk_out, constraint_out = await asyncio.gather(
            repo_task,
            risk_task,
            constraint_task,
        )

        merged = await self.merger.run(
            ctx.merge_pack(
                repo_out=repo_out,
                risk_out=risk_out,
                constraint_out=constraint_out,
            )
        )

        PlanQualityGate.validate_or_raise(merged)

        return PlanBundle(
            repo=repo_out,
            risk=risk_out,
            constraint=constraint_out,
            merged=merged,
        )

6. app/review/orchestrator.py

Claude Code Review の構造は、各 agent が別の種類の問題を見る → verification step で false positive を落とす → dedupe / severity ranking → inline comment です。これを local harness に落とします。

from __future__ import annotations

import asyncio
from dataclasses import dataclass

from app.contracts.artifact_models import ReviewFindingsV1, ReviewFinding


@dataclass(slots=True)
class ReviewBundle:
    all_findings: ReviewFindingsV1
    postable_findings: ReviewFindingsV1


class ReviewOrchestrator:
    def __init__(
        self,
        diff_reviewer,
        history_reviewer,
        contract_reviewer,
        test_reviewer,
        evidence_verifier,
        ranker,
    ) -> None:
        self.diff_reviewer = diff_reviewer
        self.history_reviewer = history_reviewer
        self.contract_reviewer = contract_reviewer
        self.test_reviewer = test_reviewer
        self.evidence_verifier = evidence_verifier
        self.ranker = ranker

    async def run(self, ctx) -> ReviewBundle:
        raw_batches = await asyncio.gather(
            self.diff_reviewer.run(ctx),
            self.history_reviewer.run(ctx),
            self.contract_reviewer.run(ctx),
            self.test_reviewer.run(ctx),
        )

        merged = self._dedupe(raw_batches)
        verified = await self.evidence_verifier.run(ctx, merged)
        ranked = await self.ranker.run(ctx, verified)

        postable = ReviewFindingsV1(
            findings=[
                f
                for f in ranked.findings
                if f.confidence >= ctx.thresholds.min_confidence_to_report
                and f.verifier_status == "confirmed"
            ]
        )

        return ReviewBundle(all_findings=ranked, postable_findings=postable)

    def _dedupe(self, batches: list[ReviewFindingsV1]) -> ReviewFindingsV1:
        seen: set[tuple[str, int, int, str]] = set()
        out: list[ReviewFinding] = []
        for batch in batches:
            for finding in batch.findings:
                key = (
                    finding.file,
                    finding.line_start,
                    finding.line_end,
                    finding.claim,
                )
                if key in seen:
                    continue
                seen.add(key)
                out.append(finding)
        return ReviewFindingsV1(findings=out)

GitHub inline 投稿は、最初は confidence >= 0.80 かつ verifier_status == confirmed の finding だけで十分です。threshold を設定ファイルに持たせたので、synthetic eval を見ながら 0.80 → 0.85 に上げるかは後で調整できます。

4. app/workflow_loader.py を parse+validate に置き換える

現状は loader が軽いので、ここで front matter を読み、typed schema に変換し、last-known-good を持てるようにします。いまの repo でも WORKFLOW.md が repo-owned contract なので、ここを厚くするのが自然です。

from __future__ import annotations

from dataclasses import dataclass
from pathlib import Path
import re
from typing import Any

import yaml
from pydantic import ValidationError

from app.contracts.workflow_schema import WorkflowConfig


_FRONT_MATTER_RE = re.compile(
    r"^---\n(?P<yaml>.*?)\n---\n(?P<body>.*)$",
    re.DOTALL,
)


@dataclass(slots=True)
class WorkflowDefinition:
    path: Path
    raw_text: str
    raw_config: dict[str, Any]
    config: WorkflowConfig
    prompt_body: str


class WorkflowLoadError(RuntimeError):
    pass


class WorkflowValidationError(RuntimeError):
    pass


def load_workflow(path: str | Path) -> WorkflowDefinition:
    workflow_path = Path(path)
    text = workflow_path.read_text(encoding="utf-8")

    match = _FRONT_MATTER_RE.match(text)
    if not match:
        raise WorkflowLoadError("WORKFLOW.md must start with YAML front matter")

    raw_config = yaml.safe_load(match.group("yaml")) or {}
    body = match.group("body").strip()

    if not isinstance(raw_config, dict):
        raise WorkflowLoadError("workflow front matter must decode to a mapping")

    try:
        config = WorkflowConfig.model_validate(raw_config)
    except ValidationError as exc:
        raise WorkflowValidationError(str(exc)) from exc

    return WorkflowDefinition(
        path=workflow_path,
        raw_text=text,
        raw_config=raw_config,
        config=config,
        prompt_body=body,
    )

Python Agent SDK では query() が new session each time、ClaudeSDKClient が continuous conversation 向けです。committee の前3役は query()、差し戻しや plan revision を継続させたいときだけ ClaudeSDKClient に寄せるのがきれいです。setting_sources=["project"] を省くと filesystem settings を読まず、project の CLAUDE.md を読むには setting_sources=["project"] が必要です。output_format には JSON Schema を渡せます。

from __future__ import annotations

import asyncio
from dataclasses import dataclass

from app.contracts.artifact_models import (
    ConstraintReportV1,
    PlanV2,
    RepoExplorerV1,
    RiskTestPlanV1,
)
from app.planning.context_builder import CommitteeContextBuilder
from app.planning.plan_quality_gate import PlanQualityGate
from app.planning.roles.constraint_checker import ConstraintChecker
from app.planning.roles.plan_merger import PlanMerger
from app.planning.roles.repo_explorer import RepoExplorer
from app.planning.roles.risk_test_planner import RiskTestPlanner


@dataclass(slots=True)
class PlanBundle:
    repo: RepoExplorerV1
    risk: RiskTestPlanV1
    constraint: ConstraintReportV1
    merged: PlanV2


class PlannerCommittee:
    def __init__(
        self,
        repo_explorer: RepoExplorer,
        risk_test_planner: RiskTestPlanner,
        constraint_checker: ConstraintChecker,
        merger: PlanMerger,
    ) -> None:
        self.repo_explorer = repo_explorer
        self.risk_test_planner = risk_test_planner
        self.constraint_checker = constraint_checker
        self.merger = merger

    async def build_plan(self, issue_ctx) -> PlanBundle:
        ctx = CommitteeContextBuilder.from_issue(issue_ctx)

        repo_task = self.repo_explorer.run(ctx.repo_pack)
        risk_task = self.risk_test_planner.run(ctx.risk_pack)
        constraint_task = self.constraint_checker.run(ctx.constraint_pack)

        repo_out, risk_out, constraint_out = await asyncio.gather(
            repo_task,
            risk_task,
            constraint_task,
        )

        merged = await self.merger.run(
            ctx.merge_pack(
                repo_out=repo_out,
                risk_out=risk_out,
                constraint_out=constraint_out,
            )
        )

        PlanQualityGate.validate_or_raise(merged)

        return PlanBundle(
            repo=repo_out,
            risk=risk_out,
            constraint=constraint_out,
            merged=merged,
        )

            raise ValueError("candidate_files required")
        if not plan.verification_profile:
            raise ValueError("verification_profile required")
        if not plan.test_mapping:
            raise ValueError("test_mapping required")
        mapped = {item.criterion for item in plan.test_mapping}
        missing = [c for c in plan.acceptance_criteria if c not in mapped]
        if missing:
            raise ValueError(f"unmapped acceptance criteria: {missing}")
        if not plan.design_branches:
            raise ValueError("design_branches required")

    @staticmethod
    def should_enable_candidate_mode(plan: PlanV2, rework_count: int) -> bool:

            (rework_count >= 1 or plan.planner_confidence < 0.75)
            and len(plan.design_branches) >= 2

Claude Code Review の managed 版は、specialized agents が full codebase context で PR を見て、severity を付けた inline findings を返します。CLAUDE.md と REVIEW.md で what to flag / skip を調整できます。公式 plugin README でも、ローカル plugin 版は 4 並列 agent、confidence 80 threshold、terminal 出力既定、--comment で投稿という構成です。これを local harness に落とすのが今回の review lane です。

from __future__ import annotations

import asyncio
from dataclasses import dataclass

from app.contracts.artifact_models import ReviewFinding, ReviewFindingsV1


@dataclass(slots=True)
class ReviewBundle:
    all_findings: ReviewFindingsV1
    postable_findings: ReviewFindingsV1


class ReviewOrchestrator:
    def __init__(
        self,
        diff_reviewer,
        history_reviewer,
        contract_reviewer,
        test_reviewer,
        evidence_verifier,
        ranker,
    ) -> None:
        self.diff_reviewer = diff_reviewer
        self.history_reviewer = history_reviewer
        self.contract_reviewer = contract_reviewer
        self.test_reviewer = test_reviewer
        self.evidence_verifier = evidence_verifier
        self.ranker = ranker

    async def run(self, ctx) -> ReviewBundle:
        raw_batches = await asyncio.gather(
            self.diff_reviewer.run(ctx),
            self.history_reviewer.run(ctx),
            self.contract_reviewer.run(ctx),
            self.test_reviewer.run(ctx),
        )

        merged = self._dedupe(raw_batches)
        verified = await self.evidence_verifier.run(ctx, merged)
        ranked = await self.ranker.run(ctx, verified)

        postable = ReviewFindingsV1(
            findings=[
                f
                for f in ranked.findings
                if f.confidence >= ctx.thresholds.min_confidence_to_report
                and f.verifier_status == "confirmed"
            ]
        )

        return ReviewBundle(
            all_findings=ranked,
            postable_findings=postable,
        )

    def _dedupe(self, batches: list[ReviewFindingsV1]) -> ReviewFindingsV1:
        seen: set[tuple[str, int, int, str]] = set()
        out: list[ReviewFinding] = []

        for batch in batches:
            for finding in batch.findings:
                key = (
                    finding.file,
                    finding.line_start,
                    finding.line_end,
                    finding.claim,
                )
                if key in seen:
                    continue
                seen.add(key)
                out.append(finding)

        return ReviewFindingsV1(findings=out)

def select_winner(primary, alt1) -> str:
    def score(c) -> int:
        s = 0
        s += 100 if c.verification.hard_checks_pass else -1000
        s -= 50 * c.review.high_count
        s -= 20 * c.review.medium_count
        s -= 10 * c.scope.unexpected_file_count
        s += 10 * c.summary.plan_alignment_score
        return s

    return max([(score(primary), "primary"), (score(alt1), "alt1")])[1]

from __future__ import annotations

import asyncio
import json
from pathlib import Path

from app.runners.execution_backend import ExecutionBackend, RunArtifacts, RunHandle, RunSpec


class CodexAppServerBackend(ExecutionBackend):
    def __init__(self, command: str = "codex app-server") -> None:
        self.command = command
        self._proc = None
        self._stdout_task = None
        self._event_log_path: Path | None = None

    async def start_run(self, spec: RunSpec) -> RunHandle:
        self._event_log_path = Path(spec.cwd).parent / "raw_codex_events.jsonl"

        self._proc = await asyncio.create_subprocess_shell(
            self.command,
            cwd=spec.cwd,
            stdin=asyncio.subprocess.PIPE,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )

        await self._send(
            {
                "id": 1,
                "method": "initialize",
                "params": {
                    "clientInfo": {"name": "dev-bot", "version": "phase1"},
                    "capabilities": {},
                },
            }
        )
        await self._await_response(1)

        await self._send({"method": "initialized", "params": {}})

        await self._send(
            {
                "id": 2,
                "method": "thread/start",
                "params": {
                    "model": spec.model,
                    "cwd": spec.cwd,
                    "approvalPolicy": "never",
                    "sandbox": "workspaceWrite",
                    "serviceName": spec.service_name,
                },
            }
        )
        thread_result = await self._await_response(2)
        thread_id = thread_result["result"]["thread"]["id"]

        await self._send(
            {
                "id": 3,
                "method": "turn/start",
                "params": {
                    "threadId": thread_id,
                    "input": [{"type": "text", "text": spec.prompt}],
                    "cwd": spec.cwd,
                    "approvalPolicy": "never",
                    "model": spec.model,
                    "effort": "medium",
                    "summary": "concise",
                    "sandboxPolicy": {
                        "type": "workspaceWrite",
                        "writableRoots": spec.writable_roots or [spec.cwd],
                        "readOnlyAccess": {
                            "type": "restricted",
                            "includePlatformDefaults": True,
                            "readableRoots": spec.read_only_roots,
                        },
                        "networkAccess": spec.network_access,
                    },
                    "outputSchema": self._load_output_schema(spec.output_schema_name),
                },
            }
        )
        turn_result = await self._await_response(3)
        turn_id = turn_result["result"]["turn"]["id"]

        self._stdout_task = asyncio.create_task(self._pump_stdout())
        return RunHandle(thread_id=thread_id, turn_id=turn_id, process_id=self._proc.pid)

    async def steer(self, handle: RunHandle, message: str) -> None:
        await self._send(
            {
                "id": 31,
                "method": "turn/steer",
                "params": {
                    "threadId": handle.thread_id,
                    "expectedTurnId": handle.turn_id,
                    "input": [{"type": "text", "text": message}],
                },
            }
        )
        await self._await_response(31)

    async def interrupt(self, handle: RunHandle) -> None:
        await self._send(
            {
                "id": 32,
                "method": "turn/interrupt",
                "params": {
                    "threadId": handle.thread_id,
                    "turnId": handle.turn_id,
                },
            }
        )
        await self._await_response(32)

    async def resume_same_run(self, handle: RunHandle) -> RunHandle:
        await self._send(
            {
                "id": 41,
                "method": "thread/resume",
                "params": {
                    "threadId": handle.thread_id,
                },
            }
        )
        result = await self._await_response(41)
        thread_id = result["result"]["thread"]["id"]
        return RunHandle(thread_id=thread_id, turn_id=handle.turn_id, process_id=self._proc.pid)

    async def collect_outputs(self, handle: RunHandle) -> RunArtifacts:
        impl_path = "artifacts/implementation_result.json"
        return RunArtifacts(
            implementation_result_path=impl_path,
            raw_event_log_path=str(self._event_log_path) if self._event_log_path else "",
        )

    async def _pump_stdout(self) -> None:
        assert self._proc and self._proc.stdout and self._event_log_path
        with self._event_log_path.open("a", encoding="utf-8") as fp:
            while True:
                line = await self._proc.stdout.readline()
                if not line:
                    break
                fp.write(line.decode("utf-8"))

    async def _send(self, message: dict) -> None:
        assert self._proc and self._proc.stdin
        self._proc.stdin.write((json.dumps(message) + "\n").encode("utf-8"))
        await self._proc.stdin.drain()

    async def _await_response(self, request_id: int) -> dict:
        assert self._proc and self._proc.stdout
        while True:
            line = await self._proc.stdout.readline()
            if not line:
                raise RuntimeError("codex app-server closed stdout")
            payload = json.loads(line.decode("utf-8"))
            if payload.get("id") == request_id:
                return payload
            await self._append_event(payload)

    async def _append_event(self, payload: dict) -> None:
        if not self._event_log_path:
            return
        with self._event_log_path.open("a", encoding="utf-8") as fp:
            fp.write(json.dumps(payload, ensure_ascii=False) + "\n")

    def _load_output_schema(self, name: str) -> dict:
        if name == "implementation_result_v1":
            return {
                "type": "object",
                "properties": {
                    "candidate_id": {"type": "string"},
                    "summary": {"type": "string"},
                    "changed_files": {"type": "array", "items": {"type": "string"}},
                    "tests_run": {"type": "array", "items": {"type": "string"}},
                    "followups": {"type": "array", "items": {"type": "string"}},
                    "blocked_reasons": {"type": "array", "items": {"type": "string"}},
                },
                "required": ["candidate_id", "summary", "changed_files"],
                "additionalProperties": False,
            }
        raise KeyError(name)

from __future__ import annotations

from dataclasses import dataclass
from pathlib import Path
from typing import Any
import re

import yaml
from pydantic import ValidationError

from app.contracts.workflow_schema import WorkflowConfig


_FRONT_MATTER_RE = re.compile(
    r"^---\n(?P<yaml>.*?)\n---\n(?P<body>.*)$",
    re.DOTALL,
)


@dataclass(slots=True)
class WorkflowDefinition:
    path: Path
    raw_text: str
    raw_config: dict[str, Any]
    config: WorkflowConfig
    prompt_body: str


class WorkflowLoadError(RuntimeError):
    pass


class WorkflowValidationError(RuntimeError):
    pass


def _resolve_root(workspace: str | None = None, repo_root: str | None = None) -> Path:
    return Path(workspace or repo_root or ".").resolve()


def load_workflow_definition(
    workspace: str | None = None,
    repo_root: str | None = None,
) -> WorkflowDefinition | None:
    root = _resolve_root(workspace=workspace, repo_root=repo_root)
    path = root / "WORKFLOW.md"
    if not path.exists():
        return None

    text = path.read_text(encoding="utf-8")
    match = _FRONT_MATTER_RE.match(text)
    if not match:
        raise WorkflowLoadError("WORKFLOW.md must begin with YAML front matter")

    raw_config = yaml.safe_load(match.group("yaml")) or {}
    if not isinstance(raw_config, dict):
        raise WorkflowLoadError("workflow front matter must decode to a mapping")

    prompt_body = match.group("body").strip()

    try:
        config = WorkflowConfig.model_validate(raw_config)
    except ValidationError as exc:
        raise WorkflowValidationError(str(exc)) from exc

    return WorkflowDefinition(
        path=path,
        raw_text=text,
        raw_config=raw_config,
        config=config,
        prompt_body=prompt_body,
    )


def load_workflow(
    workspace: str | None = None,
    repo_root: str | None = None,
) -> dict[str, Any]:
    definition = load_workflow_definition(workspace=workspace, repo_root=repo_root)
    if definition is None:
        return {}

    payload = dict(definition.raw_config)
    payload["raw_text"] = definition.raw_text
    payload["contract_body"] = definition.prompt_body
    payload["config"] = definition.config
    payload["workflow_definition"] = definition
    return payload


def workflow_text(
    workspace: str | None = None,
    repo_root: str | None = None,
) -> str:
    payload = load_workflow(workspace=workspace, repo_root=repo_root)
    return str(payload.get("raw_text", ""))

class PlanQualityGate:
    @staticmethod
    def validate_or_raise(plan: PlanV2) -> None:
        if not plan.acceptance_criteria:
            raise ValueError("acceptance_criteria required")
        if plan.out_of_scope is None:
            raise ValueError("out_of_scope field required")
        if not plan.candidate_files:
            raise ValueError("candidate_files required")
        if not plan.verification_profile:
            raise ValueError("verification_profile required")
        if not plan.test_mapping:
            raise ValueError("test_mapping required")

        mapped = {item.criterion for item in plan.test_mapping}
        missing = [c for c in plan.acceptance_criteria if c not in mapped]
        if missing:
            raise ValueError(f"unmapped acceptance criteria: {missing}")

        if not plan.design_branches:
            raise ValueError("design_branches required")

    @staticmethod
    def should_enable_candidate_mode(plan: PlanV2, rework_count: int) -> bool:
        return (
            (rework_count >= 1 or plan.planner_confidence < 0.75)
            and len(plan.design_branches) >= 2
        )

@dataclass(slots=True)
class PlanBundle:
    repo: RepoExplorerV1
    risk: RiskTestPlanV1
    constraint: ConstraintReportV1
    merged: PlanV2


class PlannerCommittee:
    def __init__(
        self,
        repo_explorer: RepoExplorer,
        risk_test_planner: RiskTestPlanner,
        constraint_checker: ConstraintChecker,
        merger: PlanMerger,
    ) -> None:
        self.repo_explorer = repo_explorer
        self.risk_test_planner = risk_test_planner
        self.constraint_checker = constraint_checker
        self.merger = merger

    async def build_plan(self, issue_ctx) -> PlanBundle:
        ctx = CommitteeContextBuilder.from_issue(issue_ctx)

        repo_task = self.repo_explorer.run(ctx.repo_pack)
        risk_task = self.risk_test_planner.run(ctx.risk_pack)
        constraint_task = self.constraint_checker.run(ctx.constraint_pack)

        repo_out, risk_out, constraint_out = await asyncio.gather(
            repo_task,
            risk_task,
            constraint_task,
        )

        merged = await self.merger.run(
            ctx.merge_pack(
                repo_out=repo_out,
                risk_out=risk_out,
                constraint_out=constraint_out,
            )
        )

        PlanQualityGate.validate_or_raise(merged)

        return PlanBundle(
            repo=repo_out,
            risk=risk_out,
            constraint=constraint_out,
            merged=merged,
        )

from __future__ import annotations

import asyncio
from dataclasses import dataclass

from app.contracts.artifact_models import ReviewFinding, ReviewFindingsV1


@dataclass(slots=True)
class ReviewBundle:
    all_findings: ReviewFindingsV1
    postable_findings: ReviewFindingsV1


class ReviewOrchestrator:
    def __init__(
        self,
        diff_reviewer,
        history_reviewer,
        contract_reviewer,
        test_reviewer,
        evidence_verifier,
        ranker,
    ) -> None:
        self.diff_reviewer = diff_reviewer
        self.history_reviewer = history_reviewer
        self.contract_reviewer = contract_reviewer
        self.test_reviewer = test_reviewer
        self.evidence_verifier = evidence_verifier
        self.ranker = ranker

    async def run(self, ctx) -> ReviewBundle:
        raw_batches = await asyncio.gather(
            self.diff_reviewer.run(ctx),
            self.history_reviewer.run(ctx),
            self.contract_reviewer.run(ctx),
            self.test_reviewer.run(ctx),
        )

        merged = self._dedupe(raw_batches)
        verified = await self.evidence_verifier.run(ctx, merged)
        ranked = await self.ranker.run(ctx, verified)

        postable = ReviewFindingsV1(
            findings=[
                f
                for f in ranked.findings
                if f.confidence >= ctx.thresholds.min_confidence_to_report
                and f.verifier_status == "confirmed"
            ]
        )

        return ReviewBundle(
            all_findings=ranked,
            postable_findings=postable,
        )

    def _dedupe(self, batches: list[ReviewFindingsV1]) -> ReviewFindingsV1:
        seen: set[tuple[str, int, int, str]] = set()
        out: list[ReviewFinding] = []

        for batch in batches:
            for finding in batch.findings:
                key = (
                    finding.file,
                    finding.line_start,
                    finding.line_end,
                    finding.claim,
                )
                if key in seen:
                    continue
                seen.add(key)
                out.append(finding)

        return ReviewFindingsV1(findings=out)

    if should_enable_candidate_mode(plan, rework_count):
        return CandidateDecision(enabled=True, candidate_ids=["primary", "alt1"])

def select_winner(primary, alt1) -> str:
    def score(c) -> int:
        s = 0
        s += 100 if c.verification.hard_checks_pass else -1000
        s -= 50 * c.review.high_count
        s -= 20 * c.review.medium_count
        s -= 10 * c.scope.unexpected_file_count
        s += 10 * c.summary.plan_alignment_score
        return s

    return max([(score(primary), "primary"), (score(alt1), "alt1")])[1]

import asyncio

from app.runners.execution_backend import ExecutionBackend, RunArtifacts, RunHandle, RunSpec


class CodexAppServerBackend(ExecutionBackend):
    def __init__(self, command: str = "codex app-server") -> None:
        self.command = command
        self._proc: asyncio.subprocess.Process | None = None
        self._pending: dict[int, asyncio.Future] = {}
        self._reader_task: asyncio.Task | None = None
        self._event_log_path: Path | None = None
        self._request_id = 100

    async def start_run(self, spec: RunSpec) -> RunHandle:
        self._event_log_path = Path(spec.cwd).parent / "raw_codex_events.jsonl"

        self._proc = await asyncio.create_subprocess_shell(
            self.command,
            cwd=spec.cwd,
            stdin=asyncio.subprocess.PIPE,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        self._reader_task = asyncio.create_task(self._reader_loop())

        await self._request(
            "initialize",
            {
                "clientInfo": {"name": "dev-bot", "version": "phase1"},
                "capabilities": {},
            },
            request_id=1,
        )
        await self._notify("initialized", {})

        thread_resp = await self._request(
            "thread/start",
            {
                "model": spec.model,
                "cwd": spec.cwd,
                "approvalPolicy": "never",
                "sandbox": "workspaceWrite",
                "serviceName": spec.service_name,
            },
            request_id=2,
        )
        thread_id = thread_resp["result"]["thread"]["id"]

        turn_resp = await self._request(
            "turn/start",
            {
                "threadId": thread_id,
                "input": [{"type": "text", "text": spec.prompt}],
                "cwd": spec.cwd,
                "approvalPolicy": "never",
                "model": spec.model,
                "effort": "medium",
                "summary": "concise",
                "sandboxPolicy": {
                    "type": "workspaceWrite",
                    "writableRoots": spec.writable_roots or [spec.cwd],
                    "readOnlyAccess": {
                        "type": "restricted",
                        "includePlatformDefaults": True,
                        "readableRoots": spec.read_only_roots,
                    },
                    "networkAccess": spec.network_access,
                },
                "outputSchema": self._load_output_schema(spec.output_schema_name),
            },
            request_id=3,
        )
        turn_id = turn_resp["result"]["turn"]["id"]

        return RunHandle(
            thread_id=thread_id,
            turn_id=turn_id,
            process_id=self._proc.pid if self._proc else None,
        )

    async def steer(self, handle: RunHandle, message: str) -> None:
        await self._request(
            "turn/steer",
            {
                "threadId": handle.thread_id,
                "expectedTurnId": handle.turn_id,
                "input": [{"type": "text", "text": message}],
            },
        )

    async def interrupt(self, handle: RunHandle) -> None:
        await self._request(
            "turn/interrupt",
            {
                "threadId": handle.thread_id,
                "turnId": handle.turn_id,
            },
        )

    async def resume_same_run(self, handle: RunHandle) -> RunHandle:
        resp = await self._request(
            "thread/resume",
            {
                "threadId": handle.thread_id,
            },
        )
        return RunHandle(
            thread_id=resp["result"]["thread"]["id"],
            turn_id=handle.turn_id,
            process_id=self._proc.pid if self._proc else None,
        )

    async def collect_outputs(self, handle: RunHandle) -> RunArtifacts:
        impl_path = "artifacts/implementation_result.json"
        return RunArtifacts(
            implementation_result_path=impl_path,
            raw_event_log_path=str(self._event_log_path) if self._event_log_path else "",
        )

    async def _reader_loop(self) -> None:
        assert self._proc and self._proc.stdout
        while True:
            line = await self._proc.stdout.readline()
            if not line:
                break
            payload = json.loads(line.decode("utf-8"))
            await self._append_event(payload)

            req_id = payload.get("id")
            if req_id is not None and req_id in self._pending:
                future = self._pending.pop(req_id)
                if not future.done():
                    future.set_result(payload)

    async def _append_event(self, payload: dict) -> None:
        if not self._event_log_path:
            return
        with self._event_log_path.open("a", encoding="utf-8") as fp:
            fp.write(json.dumps(payload, ensure_ascii=False) + "\n")

    async def _request(
        self,
        method: str,
        params: dict,
        request_id: int | None = None,
    ) -> dict:
        rid = request_id or self._next_request_id()
        loop = asyncio.get_running_loop()
        future: asyncio.Future = loop.create_future()
        self._pending[rid] = future
        await self._write_json({"id": rid, "method": method, "params": params})
        return await future

    async def _notify(self, method: str, params: dict) -> None:
        await self._write_json({"method": method, "params": params})

    async def _write_json(self, payload: dict) -> None:
        assert self._proc and self._proc.stdin
        self._proc.stdin.write((json.dumps(payload) + "\n").encode("utf-8"))
        await self._proc.stdin.drain()

    def _next_request_id(self) -> int:
        self._request_id += 1
        return self._request_id

    def _load_output_schema(self, name: str) -> dict:
        if name == "implementation_result_v1":
            return {
                "type": "object",
                "properties": {
                    "candidate_id": {"type": "string"},
                    "summary": {"type": "string"},
                    "changed_files": {"type": "array", "items": {"type": "string"}},
                    "tests_run": {"type": "array", "items": {"type": "string"}},
                    "followups": {"type": "array", "items": {"type": "string"}},
                    "blocked_reasons": {"type": "array", "items": {"type": "string"}},
                },
                "required": ["candidate_id", "summary", "changed_files"],
                "additionalProperties": False,
            }
        raise KeyError(name)

            path = bundle_dir / filename
            if isinstance(content, (dict, list)):
                path.write_text(
                    json.dumps(content, ensure_ascii=False, indent=2),
                    encoding="utf-8",
                )
            else:
                path.write_text(str(content), encoding="utf-8")

        (bundle_dir / "manifest.json").write_text(
            json.dumps(manifest, ensure_ascii=False, indent=2),
            encoding="utf-8",
        )
        return bundle_dir

    def freeze(self, bundle_dir: Path) -> None:
        frozen_marker = bundle_dir / ".frozen"

        manifest = bundle_dir / "manifest.json"

from __future__ import annotations

from pathlib import Path
import hashlib
import yaml

ROOT = Path(__file__).resolve().parents[1]


def load_manifest() -> dict:
    return yaml.safe_load((ROOT / "docs/policy/manifest.yaml").read_text(encoding="utf-8"))


def build_output(output_name: str, sources: list[str]) -> str:
    chunks: list[str] = []
    for rel in sources:
        text = (ROOT / rel).read_text(encoding="utf-8").rstrip()
        chunks.append(text)

    body = "\n\n".join(chunks).rstrip() + "\n"
    checksum = hashlib.sha256(body.encode("utf-8")).hexdigest()

    header = [
        "<!-- GENERATED FILE. DO NOT EDIT.",
        f"output: {output_name}",
        "sources:",
        *[f"  - {src}" for src in sources],
        f"checksum: sha256:{checksum}",
        "-->",
        "",
    ]
    return "\n".join(header) + body


def main() -> None:
    manifest = load_manifest()
    for output_name, sources in manifest["outputs"].items():
        rendered = build_output(output_name, sources)
        (ROOT / output_name).write_text(rendered, encoding="utf-8")


if __name__ == "__main__":
    main()

既存コードに合わせた運用上の注意

1つ目。scheduler gate は今すぐ harden しすぎないことです。今の IssueScheduler は planning artifacts が揃わないと dispatch しないので、gate を急に厳しくすると rollout 初日に既存 issue が止まります。まずは verification_plan.json を recommended にしてログを取り、planner committee が安定してから required に上げるのがよいです。

2つ目。candidate mode の情報は WorkItem に積まないことです。今の orchestrator は WorkItem の key と queue state で重複防止しているので、ここに candidate_id を混ぜると enqueue semantics まで変わります。Phase 1 は single-writer default を保ち、candidate decision と winner selection は runtime meta / execution artifacts / state store 側で扱う方が安全です。これは Codex の multi-agent docs が、parallel write-heavy workflows に慎重であるべきだとしているのとも整合します。

3つ目。policy compile は deterministic generator のみにすることです。OpenAI は AGENTS.md を durable guidance として小さく保ち、skills や docs に分割することを勧めています。Claude review 側も REVIEW.md を分ける前提なので、docs/policy/core.md と overlays から AGENTS.md / CLAUDE.md / REVIEW.md を決定的に生成する設計はかなり筋が良いです。
